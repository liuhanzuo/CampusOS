# 登录过期问题解决方案

## 🚀 问题描述

如果你遇到了 `"登录过期，请重新登录"` (错误代码: 1130002) 的问题，系统已经自动处理了！

## ✅ 已实现的解决方案

### 1. 自动登录过期检测
- 系统会自动检测登录状态
- 发现登录过期会自动清除旧cookies
- 自动触发重新登录流程

### 2. 增强的2FA处理
- 支持企业微信验证码
- 支持手机短信验证
- 支持TOTP验证器
- 自动处理验证码输入

### 3. 智能重试机制
- 查询/预约时自动检查登录状态
- 登录过期自动重新认证
- 无需手动干预

## 🔧 使用方法

### 正常使用（推荐）

```bash
cd f:/Campusos/apps/thu-ai-assistant
npm run sports
```

登录一次后，系统会自动处理所有登录过期问题：
- ✅ 首次登录输入验证码
- ✅ 后续操作自动处理过期
- ✅ 查询/预约无需重新登录

### 手动强制重新登录

如果遇到问题，可以在主菜单选择 `4. 重新登录`：

```
📋 主菜单：
  1. 查询场馆
  2. 预约场地
  3. 查看场馆列表
  4. 重新登录    ← 选择这个
  5. 退出

请选择操作（输入数字）：4
```

### 清除Cookies重新登录

如果自动重试失败，可以手动清除cookies：

```bash
# 删除cookies文件
rm -f apps/thu-ai-assistant/.cookies/sports-cookies.json

# 然后重新运行
npm run sports
```

## 💡 工作原理

### 自动登录过期检测流程

1. **操作前检查**：每次查询/预约前检查登录状态
2. **API验证**：访问需要认证的API端点验证状态
3. **错误识别**：检测"登录过期"、"1130002"等错误信息
4. **自动清理**：清除过期cookies
5. **重新登录**：使用保存的凭据自动登录
6. **继续操作**：登录成功后继续原操作

### 2FA自动处理

1. **检测2FA要求**：识别页面中的2FA元素
2. **选择验证方式**：默认使用企业微信
3. **获取验证码**：提示用户输入验证码
4. **自动输入**：自动填充验证码到正确位置
5. **提交验证**：自动提交验证请求
6. **信任设备**：可选择信任设备30天免验证

## 🔍 故障排除

### 问题1：频繁要求重新登录

**解决方案**：
1. 在2FA时选择"信任设备"
2. 检查网络连接是否稳定
3. 避免频繁操作触发风控

### 问题2：验证码输入失败

**解决方案**：
1. 确保验证码输入正确（6位数字）
2. 检查验证码是否过期（通常60秒）
3. 尝试重新获取验证码

### 问题3：自动登录失败

**解决方案**：
1. 手动删除cookies文件
2. 使用"重新登录"功能
3. 检查账号密码是否正确

## 📞 技术支持

如果问题持续存在，请检查：

1. **浏览器驱动**：确保ChromeDriver版本匹配
2. **网络连接**：确保能访问清华体育系统
3. **账号状态**：确认清华账号正常
4. **系统时间**：确保系统时间准确
